ADB マネージャー、 あなたのワンストップによって、アンドロイドでもっと簡単に開発できる

警告: ルートが必要です ！

もしこのアプリケーションに興味がありましたら、次より翻訳のお手伝いをして下さい:

https://crowdin.net/project/adbm

最も自動化され、使いやすく、安定した、充実のサポートつきのADB 管理ツール

機能:
+ 通知バーからの簡単なコントロールとアクセスの詳細
+ 保存された WiFi ネットワークに自動接続します。
+ ブート時に自動起動させるか、させないかを設定画面から選択できます
+ WiFi から切断、またはWiFi に接続するとき、自動的にUSBモードかネットワークモードに切り替える
+ 設定可能なサービスの管理
+ ADB の状態に応じて異なるカラーコードのアイコン 
+ サービスを実行中は画面をオンにしておく
+ 新しいパッケージをインストールする時は画面を起こす
+ 通知バーの画像に触れることによってADB ステートを切り替えることができます

通知バーでいつでも見ることができるので、他のものより本当にもっと簡単に使用できる WiFi ネットワークと、ネットワークの状態に基づいて、 ADB 状態のコントロールを完全に自動化するような高度な機能をサポートします。

プリファランスのリストから既知の設定されたネットワークにつないだ場合、ADB マネージャーは自動的にネットワークモードにてADB を開始することが可能です。

WiFi にて、カフェ/バー/電車/トイレ/その他の場所で開発中ですか？ ADB マネージャーが 自動的にADB からワイヤレスに切り替えたり、戻したりします。

オープンソースとしてリリースしました、次のコードでみることができます　https://github.com/ilijamt/android-adbm

将来のバージョン:
+ 状態の切り替えによる遅れ

アクセス許可:
+ android.permission.ACCESS_SUPERUSER 
利用可能な場合 SuperSU を使用

+ android.permission.RECEIVE_BOOT_COMPLETED
デバイスのブート時に自動的に開始するために使用。開始するのにどのくらい待つ必要があるのか、どのくらいの頻度で AlarmManager がサービスの実行状況をチェックするのかは、設定メニューから設定可能です。

+ android.permission.ACCESS_NETWORK_STATE
  android.permission.ACCESS_WIFI_STATE
ADB 状態間の切り替えを自動化するために使用

+ android.permission.INTERNET
クラッシュによりこの許可なしに IP アドレスをチェックするとき、IPを回復するために使用

+ android.permission.WAKE_LOCK
新しいパッケージのインストール時にスクリーンを起こすため、またはADB サービスが起動中画面を表示したままにするために使用した

既知の問題:
API 8 バージョンでは、LDPI デバイスのメニューアイコンは少し変に見えます。
